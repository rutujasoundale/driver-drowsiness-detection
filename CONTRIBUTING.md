# Contributing to Driver Drowsiness Detection System 🚗

Thank you for your interest in contributing to this project!
This repository is open for improvements, bug fixes, and new features.

The goal of this project is to build an **AI-based Driver Drowsiness Detection System** using computer vision.

---

# How You Can Contribute

You can contribute in many ways:

• Fix bugs in the code
• Improve detection accuracy
• Add new features (example: head pose detection, sound alert)
• Improve documentation
• Optimize performance
• Add UI improvements

Even small contributions are welcome.

---

# Getting Started

### 1. Fork the Repository

Click the **Fork** button on the repository page.

This creates a copy of the project in your GitHub account.

---

### 2. Clone the Repository

Clone your forked repository to your computer.

```
git clone https://github.com/YOUR_USERNAME/driver-drowsiness-detection.git
```

Move into the project directory:

```
cd driver-drowsiness-detection
```

---

### 3. Install Dependencies

Install required Python libraries:

```
pip install -r requirements.txt
```

---

### 4. Create a New Branch

Before making changes, create a new branch.

```
git checkout -b feature-name
```

Example:

```
git checkout -b add-sound-alert
```

---

### 5. Make Your Changes

You can work on issues such as:

• Add audio alert when drowsiness is detected
• Improve yawning detection
• Add head pose detection
• Improve UI overlay

Test your code before committing.

---

### 6. Commit Your Changes

After completing the changes:

```
git add .
git commit -m "Added sound alert for drowsiness detection"
```

---

### 7. Push Changes

Push your branch to GitHub.

```
git push origin feature-name
```

---

### 8. Create a Pull Request

Go to your forked repository and click **New Pull Request**.

Explain the changes you made and submit it.

The project maintainer will review your contribution.

---

# Code Guidelines

Please follow these guidelines:

• Write clean and readable Python code
• Add comments where necessary
• Do not break existing functionality
• Test your code before submitting

---

# Reporting Bugs

If you find a bug, please open an issue and include:

• Description of the problem
• Steps to reproduce the issue
• Expected behavior
• Screenshots (if possible)
